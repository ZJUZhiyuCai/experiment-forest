# 硅基流动模型配置指南

## 🚨 解决 "模型不存在" 错误 (错误代码: 20012)

### 问题描述
当您看到以下错误时：
```
API连接测试失败
请求失败 (400)
状态信息：
详细信息：{"code":20012,"message":"Model does not exist. Please check it carefully.","data":null}
```

这表示您选择的模型在硅基流动API中不存在或不可用。

## ✅ 解决方案

### 第一步：检查当前模型配置
1. 进入系统设置页面 → AI API配置
2. 查看当前选择的模型名称
3. 确认是否使用了正确的模型名称

### 第三步：配置步骤

   - 应该看到"连接测试成功"的提示

## 🔍 故障排除

### 问题1：模型列表中没有我想要的模型
**解决方案：**
- 选择"自定义模型"选项
- 在自定义模型名称字段中输入确切的模型名称
- 确保模型名称与硅基流动支持的模型完全一致

### 问题2：仍然显示模型不存在
**可能原因：**
1. 账户权限不足
2. 模型名称拼写错误
3. 账户余额不足
4. 该模型暂时不可用

**解决步骤：**
1. 登录硅基流动控制台：https://cloud.siliconflow.cn
2. 检查账户状态和余额
3. 查看可用模型列表
4. 确认模型访问权限

### 问题3：测试成功但实际使用时仍然失败
**解决方案：**
1. 刷新浏览器页面
2. 重新保存AI设置
3. 检查浏览器控制台是否有其他错误
4. 联系硅基流动技术支持
